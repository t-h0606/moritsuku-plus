# もりつく＋ をGitHubに置いて公開するまで

ブラウザだけで完結します。コマンド操作は一切ありません。所要30分程度。

---

## 事前に用意するもの

- メールアドレス
- 解凍済みの `moritsuku-plus-v26` フォルダ

---

## ステップ1：アカウントを作る（5分）

1. `https://github.com/` を開き、右上の **Sign up** を押す
2. メールアドレス、パスワード、ユーザー名を入れる
3. メールに届いた8桁の数字を入力

**ユーザー名は後でURLの一部になります。** `moritsuku` のように短くて分かりやすいものが良いです。日本語は使えません。

例：ユーザー名を `moritsuku` にすると、サイトのURLは
`https://moritsuku.github.io/moritsuku-plus/` になります。

料金プランを聞かれたら **Free** を選んでください。今回やることはすべて無料の範囲です。

---

## ステップ2：置き場所を作る（3分）

1. 画面右上の **＋** → **New repository**
2. **Repository name** に `moritsuku-plus` と入力
3. **Public** を選ぶ（★重要。Privateだと無料では公開できません）
4. 他の項目は触らず、下の **Create repository**

> Publicは「誰でもファイルを見られる」という意味です。Webサイトとして公開する以上これで問題ありませんが、**パスワードやAPIキーなど秘密の情報を置いてはいけません**。今回のファイルには一切含まれていないので大丈夫です。

---

## ステップ3：ファイルを置く（5分）

作成直後の画面に **uploading an existing file** というリンクがあります。それを押します。

そして――ここが引っかかりやすい箇所です。

**`moritsuku-plus-v26` フォルダごとドラッグしないでください。**

フォルダごと入れると、URLが `.../moritsuku-plus-v26/index.html` のように一段深くなってしまいます。

**正しい手順：**

1. `moritsuku-plus-v26` フォルダを開く
2. その**中身**（`index.html` `css` `js` `images` `data` `icons` `manifest.json` `sw.js` など）を全部選択
3. それをブラウザの画面にドラッグ＆ドロップ

画面に `index.html` `css/style.css` … と並べば成功です。

4. 下までスクロールして **Commit changes** を押す

アップロードは1〜2分かかります。

---

## ステップ4：公開スイッチを入れる（2分）

1. 上部メニューの **Settings**
2. 左サイドバーの **Pages**
3. **Source** が `Deploy from a branch` になっていることを確認
4. **Branch** を `main` に、その右の欄を `/ (root)` にして **Save**

数分待ってページを再読み込みすると、上部にURLが表示されます。それがサイトのアドレスです。

**初回は反映まで5〜10分かかることがあります。** すぐに出なくても、お茶を淹れて戻ってきてください。

---

## ステップ5：スマホで確認する

発行されたURLをスマホで開きます。

**iPhone**：下部の共有ボタン（□に↑）→ 下にスクロール →「ホーム画面に追加」
**Android**：右上の3点メニュー →「アプリをインストール」または「ホーム画面に追加」

アイコンから開いて、アドレスバーが消えて全画面で立ち上がれば成功です。

---

## 記事を追加するとき（今後の日常作業）

1. GitHubで `data` フォルダ → `articles.json` を開く
2. 右上の **鉛筆マーク** を押す
3. 私がお渡しした内容に書き換える
4. 下の **Commit changes** → **Commit changes**

1〜2分でサイトに反映されます。

**画像を追加するとき**は、`images/thumbs` フォルダを開いて **Add file** → **Upload files** から入れてください。

---

## つまずいたときの見どころ

| 症状 | 原因として多いもの |
|---|---|
| 画像が全部出ない | フォルダごとアップロードしてしまい、階層がずれている |
| 記事が「読み込みに失敗」 | `articles.json` のカンマ抜け・カッコの閉じ忘れ |
| 変更が反映されない | 反映待ち。5分待ってから、ブラウザの再読み込み |
| 404 と出る | Pagesの設定直後。10分待つ |

`articles.json` を書き換えて画面が真っ白になったら、**編集画面で元に戻すのではなく、GitHubの履歴から前の状態に戻せます**。壊しても必ず戻せるので、恐れずに触って大丈夫です。

## 用語のかんたんな対応表

| GitHubの言葉 | 意味 |
|---|---|
| Repository（リポジトリ） | ひとつのプロジェクトの入れ物 |
| Commit（コミット） | 変更を確定して記録すること。「保存」に近い |
| main | 本番として使われる系統の名前 |
| Pages | 置いたファイルをWebサイトとして公開する機能 |
